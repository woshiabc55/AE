# Story Outline Design

## When To Use
- The user asks for "故事大纲", "剧情大纲", "剧本大纲", " synopsis", "plot outline", or "narrative structure".
- The user has a concept, theme, or logline and wants a structured outline.
- The user wants to adapt an idea into a multi-act story.

## Input To Gather
- Genre / type (film, short video, novel, stage play, game).
- Target length (short story, feature film, 1-minute video, series pilot).
- Core concept / logline (1-2 sentences).
- Theme or emotional core.
- Protagonist and antagonist (if any).
- Desired ending tone (happy, tragic, ambiguous, twist).
- Any required beats or constraints.

## Design Rules
- Always propose a clear three-act structure unless the user requests another model (e.g., Save the Cat, Hero's Journey, Kishōtenketsu, five-act).
- Each act must include: purpose, key beats, emotional shift, and stakes escalation.
- Identify the inciting incident, midpoint reversal, climax, and resolution explicitly.
- Keep the outline concise but evocative; avoid writing full scenes unless asked.
- Ensure cause-and-effect logic between beats.

## Output Format
Use the following structure:

```
# 故事大纲：{标题}

## 一句话概念
{logline}

## 主题与情感核心
{theme}

## 主要角色
- {主角}：{目标/缺陷}
- {反派/阻力}：{对立动机}

## 结构模型
{模型名称}

## 第一幕：铺垫与切入（约占 %）
- 世界观/日常状态
- 激励事件
- 关键转折：主角做出选择

## 第二幕：对抗与升级（约占 %）
- 上升行动
- 中点反转
- 低谷/濒临失败

## 第三幕：高潮与收束（约占 %）
- 最终对决/关键抉择
- 高潮
- 结局与主题回扣

## 关键情节点清单
1. ...
2. ...

## 可选扩展方向
- 支线建议
- 伏笔回收点
- 续集/系列化可能
```

## Tone
- Analytical yet creative; preserve the user's original intent and style.
- Ask clarifying questions only when required fields are missing.
