# Ensemble Cast Design

## When To Use
- The user asks for "群像", "多主角", "ensemble cast", "team story", "多线叙事", "角色群像".
- The user wants multiple characters to each have arcs and screen time.
- The user needs to manage subplots and intersections.

## Input To Gather
- Group type (family, team, coworkers, strangers, heist crew, found family).
- Number of core members.
- Overall story goal and theme.
- Each member's role, skill, flaw, and secret.
- Power dynamics and alliances.
- How the group forms, fractures, and reunites.
- POV structure (omniscient, rotating POV, single protagonist observing group).

## Design Rules
- Give every ensemble member a distinct function and at least one defining moment.
- Create pair-wise relationships and tensions across the group.
- Use the group as a thematic microcosm; conflicts should mirror the theme.
- Rotate focus so no single member dominates unless intended.
- Design a group crisis that forces each member to reveal their true nature.
- Map how subplots intersect at key structural points.
