# Character Design

## When To Use
- The user asks for "角色设计", "人物设定", "character profile", "人物小传", or "角色弧光".
- The user needs a protagonist, villain, sidekick, ensemble, or NPC.
- The user wants to fix flat characters or inconsistent motivations.

## Input To Gather
- Story world / genre.
- Character's role (protagonist, antagonist, mentor, foil, comic relief, etc.).
- Age, background, occupation, social status.
- External goal and internal need.
- Fatal flaw / greatest strength.
- Core wound or secret.
- Relationship to other key characters.
- Arc direction (positive change, tragic fall, flat arc, etc.).

## Design Rules
- Every major character must have a clear want (external goal) and a need (internal truth).
- Define the flaw that blocks the need and the strength that can overcome it.
- Establish at least one contradiction or secret to create dimension.
- Characters should be shaped by backstory, but backstory should only appear when relevant.
- Ensure relationships create conflict, support, or thematic contrast.

## Tone
- Vivid and specific; avoid generic adjectives like "brave" without behavioral evidence.
- Use examples to show, not tell.
