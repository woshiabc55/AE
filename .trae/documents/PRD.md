# 游戏官方设定图收录与可视化画廊 —— 产品需求文档

## 1. 项目背景
为二次创作（同人、设定参考、素材整理）提供一个可本地运行、可扩展的**游戏官方美术资源收录框架**。用户自行运行抓取脚本获取图片，并承担相应版权责任。本项目仅输出代码框架与可视化界面，不内置任何受版权保护的图片。

## 2. 产品目标
- 支持按游戏、分类、角色/皮肤维度整理官方设定图。
- 提供可配置的通用抓取脚本，内置王者荣耀示例。
- 提供现代化、响应式的 HTML 可视化画廊，支持搜索、筛选、灯箱预览。
- 输出结果可直接在浏览器打开，便于二次创作时快速查阅。

## 3. 用户画像
- 二次创作者、画师、游戏设定整理者。
- 具备基础 Python 运行环境，能自行配置来源站点。
- 了解并自行承担所抓取素材的版权与使用合规责任。

## 4. 功能需求

### 4.1 数据层
- 游戏清单（games.json）：名称、分类、官网/来源、抓取配置模板。
- 资源清单（assets/data.json）：由 scraper.py 生成，记录每个资源的本地路径、远程 URL、角色/皮肤名称、所属游戏、分类标签。

### 4.2 抓取层
- 通用 Python 抓取框架（scraper.py）。
- 每个游戏独立解析器，通过配置注册到框架。
- 内置王者荣耀解析器示例，支持英雄列表与皮肤列表。
- 支持断点续抓、去重、限速、自定义 User-Agent、Referer。
- 图片按 `assets/<game>/<character>/` 结构存储。

### 4.3 展示层
- 单页 HTML（index.html），无需构建工具。
- 顶部游戏分类横向筛选栏。
- 搜索框：按游戏名、角色名、皮肤名搜索。
- 瀑布流/网格画廊，响应式布局。
- 灯箱：点击缩略图查看大图，支持键盘左右切换、ESC 关闭。
- 图片懒加载与占位骨架屏。
- 暗色主题，强调色可按游戏品牌色动态调整。

## 5. 非功能需求
- 纯前端展示，无后端依赖。
- 本地化优先：图片与索引均可离线使用。
- 代码框架清晰，便于用户扩展更多游戏解析器。
- 对抓取失败、网络异常、站点结构变化有日志与跳过机制。

## 7. 设计方向
- **风格**：暗色影院感（Dark Cinema）+ 霓虹游戏 accents。
- **版式**：大字号标题、卡片悬浮效果、非对称网格、充足留白。
- **动效**：页面加载 stagger、卡片 hover 缩放与光泽扫过、灯箱平滑过渡。
- **字体**：Display 使用锐利几何无衬线，正文使用高可读性无衬线。
