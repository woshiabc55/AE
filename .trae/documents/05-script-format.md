# 05 · 剧本格式规范

## 一、概述

剧本正文（`Template.body`）是一段 **Markdown 增强文本**，支持：
1. 标准 Markdown 语法（标题、列表、代码块、强调）
2. **`{{variable}}` 变量插槽** — 核心扩展
3. 隐式约定（片段、行内结构、字数提示）

## 二、变量插槽语法

### 2.1 基本形式

```
{{variable_name}}
```

### 2.2 命名规则

| 规则 | 例子 |
|------|------|
| 必须以字母或下划线开头 | `{{name}}` `{{_internal}}` |
| 仅含字母、数字、下划线 | `{{product_v2}}` |
| 大小写敏感 | `{{Name}}` 与 `{{name}}` 不同 |
| 长度 ≤ 32 字符 | `{{target_audience}}` |
| 不允许连字符 / 空格 | ❌ `{{product-name}}` |

### 2.3 解析器实现

```ts
// src/lib/utils.ts
const re = /\{\{\s*([a-zA-Z_][a-zA-Z0-9_]*)\s*\}\}/g;

export function extractVariableKeys(text: string): string[] {
  const set = new Set<string>();
  let m;
  while ((m = re.exec(text))) set.add(m[1]);
  return [...set];
}
```

- `\s*` 容忍 `{{ name }}` 这种带空格的写法
- 自动去重
- 不区分嵌套 / 跨行（一个变量必然在同一行）

### 2.4 渲染器实现

```ts
export function renderTemplate(body: string, values: Record<string, string>): string {
  return body.replace(re, (_, key) => {
    const v = values[key];
    if (!v || !v.trim()) return `{{${key}}}`;  // 保留原样，便于视觉提示
    return v;
  });
}
```

> **设计选择**：当变量未填时，渲染结果保留 `{{key}}` 形式而不是空字符串，让最终用户看到"哪些位置还没填"，避免静默 bug。

## 三、Markdown 风格约定

### 3.1 标题层级

| 层级 | 用法 | 例子 |
|------|------|------|
| `#` | 模块名（角色 / 任务 / 输出要求） | `# 角色` |
| `##` | 段落标题 | `## 钩子结构` |
| `###` | 子段落 | `### 第 1 段：钩子共鸣` |

### 3.2 列表

有序列表用于流程：
```
1. **前 3 秒反差钩**
2. **第 4-10 秒痛点放大**
3. **第 11-30 秒价值递进 + 逼单**
```

无序列表用于并列项：
```
- 段 1：心流不是天赋是训练
- 段 2：环境设计 > 意志力
- 段 3："无聊训练" 30 天实验
```

### 3.3 强调

- `**粗体**` 用于关键约束词
- `*斜体*` 极少用
- 不使用下划线（在编辑器和等宽字体中容易与变量插槽混淆）

### 3.4 代码 / 内联

- 反引号 `\`code\`` 用于**反例 / 术语 / 字面值**
- 例：`不要用"震惊体"` → 渲染为带背景的内联代码

### 3.5 引用 / 分隔

- 不使用 `>` 引用块（视觉太重）
- 章节间用空行 + 下一级标题分隔

## 四、推荐模板结构

我们调研了 16 个种子模板后总结出的**黄金结构**：

```
# 角色
[AI 扮演的角色描述，1-3 句，越具体越好]

# 任务
[核心任务，1 句话]

# 输入
[用 {{variable}} 插槽标注的输入参数，分组]

# 结构 / 输出要求
[分步骤 / 分段落的输出规范]

# 风格
[语气、句式、禁忌词的硬性要求]

# 反面示例（避免）
[1-3 个错误示范，用 ❌ 或反引号包裹]

# 现在开始
[触发词，引导 AI 直接输出]
```

### 4.1 真实例子 · 抖音爆款带货 30s

```markdown
# 角色
你是一位有 6 年经验的抖音爆款文案操盘手，擅长把"产品卖点"翻译成"用户立刻想买"的钩子句。

# 任务
为我正在销售的「{{product_name}}」撰写一条 {{duration}} 秒的口播带货脚本，
目标受众是 {{target_audience}}，他们最痛的点是 {{pain_point}}。

# 钩子结构（务必遵守）
1. **前 3 秒反差钩**：用「你以为...其实...」或「停一下！别划走！」打破滑屏惯性。
2. **第 4-{{duration}} 秒痛点放大**：把 {{pain_point}} 描述到一个具体场景。
3. **价值递进 + 逼单**：3 个利益点 + 价格锚定 + 紧迫感。

# 输出要求
- 全程口语化、像朋友在安利，不要营销腔。
- 每句话不超过 14 个字，便于竖屏字幕。
- 字数控制在 {{word_count}} 字以内。

# 反面示例（避免）
- "亲，这款产品非常好用哦～"（太广告）
- "今天给大家带来..."（太开场白）

# 现在开始
直接输出脚本，不要解释。
```

### 4.2 真实例子 · 知乎高赞答案

```markdown
# 角色
你是一位知乎高赞答主，擅长把专业知识包装成"读完忍不住点赞"的长答案。

# 任务
围绕问题「{{question}}」写一篇 {{word_count}} 字的知乎答案。

# 答案结构

## 1. 故事钩子（前 100 字）
- 用 {{protagonist}} 的真实经历切入
- 必须有一个"反常识"细节

## 2. 反共识干货（主体 60% 字数）
- 给出 {{main_argument}} 个「与多数人认知相反」的观点

## 3. 升华金句（结尾 200 字）
- 回到 {{protagonist}} 的故事
- 引导点赞/关注的话术

# 风格
- 知乎体 + 一点点"过来人"的谦逊
- 段落短，多用 1-2 行的段落
- 关键论点加粗 (**)
```

## 五、片段库规范

内置 19 个片段（`src/data/fragments.ts`），分 6 大类：

| 类别 | 数量 | 用途 |
|------|------|------|
| `opening` 开场 | 4 | 前 5 秒的钩子 |
| `character` 角色 | 3 | 人物设定、动机、口头禅 |
| `scene` 场景 | 3 | 环境、场记、转场 |
| `conflict` 冲突 | 3 | 矛盾、内心独白、对话 |
| `twist` 反转 | 2 | 认知反转、双关 |
| `closing` 收尾 | 3 | 戛然而止、行动呼吁、钩尾 |

### 5.1 片段 Schema

```ts
interface ScriptFragment {
  id: string;            // 'f_open_1'
  label: string;         // '反差开场'
  category: 'opening' | 'character' | 'scene' | 'conflict' | 'twist' | 'closing';
  body: string;          // '你以为……其实……'
  tags: string[];
}
```

### 5.2 插入行为

- **位置**：当前光标位置
- **前导**：若光标前不是行首，自动加 `\n`
- **结果**：toast 提示「已插入『{{label}}』」

## 六、版本快照规范

每次手动保存（点 `保存` 按钮），若 `body` 变化，则追加一条 `Version`：

```ts
{
  id: 'v_xxxx',
  createdAt: '2026-06-09T10:00:00.000Z',
  snapshot: '... 完整 body ...',  // 当时 body 的完整快照
  note?: '手动保存'
}
```

- 模板详情页"版本"Tab 倒序展示
- 第一条为"当前"
- v1 不做 diff 渲染，只展示 snapshot 长度 + 时间

## 七、字数与 Token

### 7.1 字数计算

```ts
rendered.length  // 字符数
```

不区分中英文（用户最直观）。未来可加 `Intl.Segmenter` 做"汉字数 + 单词数"。

### 7.2 Token 估算

```ts
function estimateTokens(text: string): number {
  if (!text) return 0;
  const en = (text.match(/[a-zA-Z0-9\s.,!?;:'"()\-]+/g) || []).join('').length;
  const zh = text.length - en;
  return Math.max(1, Math.round(en / 4 + zh / 1.5));
}
```

- 英文：~4 字符/token（GPT 系列经验值）
- 中文：~1.5 字符/token（CLT 经验值）
- 仅供 UI 参考，**不**用于 API 计费

## 八、禁忌与边界

### ❌ 不建议在剧本中出现的

1. 不可控的随机性：`随机选 1 个`、`随便写`
2. 引用外部资源：`参考 https://...`（AI 可能编造）
3. 敏感词：`种族 / 性别 / 政治`
4. 隐式时态假设：`昨天`、`明天`（除非有 `{{date}}` 变量）

### ⚠️ 注意事项

1. **变量名不要用动词**：`{{use}}` `{{do}}` 容易与正文动词冲突视觉
2. **长变量名降低可读性**：`{{target_audience_pain_point}}` → 拆成 2 个
3. **枚举值尽量短**：`{{duration}}` 取 `15/30/45/60` 而非 `15秒/30秒/...`
4. **数字与文本分开**：`{{count}} 个` 而非 `{{count}}`

## 九、与外部 AI 的对接

用户在工坊复制最终提示词后，会粘贴到：
- ChatGPT / Claude / Gemini
- 国产大模型（文心、通义、Kimi、DeepSeek）
- 内部 API / 自动化脚本

> 我们的模板应当**与模型无关**，避免：
> - 假设特定的 system message 结构
> - 假设特定的 function calling 能力
> - 假设特定的 JSON 输出

让 AI 自适应。
