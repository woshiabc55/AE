## 1. 产品概述

像素风机甲对战小游戏——一款复古像素风格的双人本地对战格斗游戏。两位玩家分别操控各自的机甲角色，通过移动、攻击、防御等操作进行实时对战，率先将对方血量清零者获胜。

- 目标用户：复古游戏爱好者、休闲对战玩家
- 核心价值：简单易上手、视觉风格鲜明、对战体验紧张刺激

## 2. 核心功能

### 2.1 用户角色

| 角色 | 操作方式 | 核心能力 |
|------|----------|----------|
| 玩家1（红方机甲） | WASD + F(攻击) + G(防御) | 移动、攻击、防御、冲刺 |
| 玩家2（蓝方机甲） | 方向键 + J(攻击) + K(防御) | 移动、攻击、防御、冲刺 |

### 2.2 功能模块

1. **战斗页面**：双人机甲对战主场景，包含血量条、能量条、计时器、胜负判定
2. **开始页面**：游戏标题、操作说明、开始按钮
3. **结算页面**：胜负结果、重新开始按钮

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 开始页面 | 标题动画 | 像素风格标题闪烁动画 |
| 开始页面 | 操作说明 | 双方键位说明面板 |
| 开始页面 | 开始按钮 | 点击进入战斗 |
| 战斗页面 | 战斗场景 | 像素风格城市废墟背景，地面平台 |
| 战斗页面 | 机甲角色 | 红蓝两方像素机甲，含待机/移动/攻击/防御/受伤动画 |
| 战斗页面 | 血量条 | 双方血量条，受击时闪烁减少 |
| 战量页面 | 能量条 | 攻击消耗能量，能量缓慢恢复 |
| 战斗页面 | 计时器 | 99秒倒计时，时间到血量多者胜 |
| 战斗页面 | 伤害数字 | 受击时弹出像素风伤害数字 |
| 战斗页面 | 特效 | 攻击火花、防御光盾、击中震屏 |

## 3. 核心流程

玩家进入开始页面，查看操作说明后开始游戏。双方在战斗页面进行实时对战，通过移动、攻击、防御等操作消耗对方血量。当一方血量归零或倒计时结束时，判定胜负并进入结算页面，可选择重新开始。

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深灰黑底色（#0a0a0a），红色（#ff2244）代表玩家1，蓝色（#2266ff）代表玩家2
- 辅助色：金色（#ffcc00）用于能量条和伤害数字，绿色（#44ff44）用于UI高亮
- 按钮风格：像素风格3D凸起按钮，带阴影和按压效果
- 字体：像素风格字体（Press Start 2P），标题24px，正文12px，HUD 16px
- 布局风格：全屏Canvas游戏画面，HUD叠加在画面上方
- 图标风格：8-bit像素图标，无圆角，硬边

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 开始页面 | 标题 | 像素字体，闪烁动画，居中 |
| 开始页面 | 操作说明 | 左右两栏分别展示P1/P2键位，像素边框面板 |
| 战斗页面 | HUD | 顶部血量条+能量条+计时器，像素风格边框 |
| 战斗页面 | 场景 | 城市废墟背景，像素地面平台，远景建筑剪影 |
| 战斗页面 | 机甲 | 32x48像素精灵，4帧动画 |
| 结算页面 | 结果文字 | 大号像素字体，闪烁动画 |

### 4.3 响应式

- 桌面优先设计，Canvas固定16:9比例（960x540逻辑分辨率），自动缩放适配屏幕
- 不支持移动端触控操作（双人键盘对战）

### 4.4 像素素材方案

所有角色和场景素材均通过代码程序化生成（Canvas绘制），不依赖外部图片资源：

- **机甲精灵**：使用像素矩阵定义，每个机甲由32x48像素网格构成，包含身体、头部、手臂、腿部等部件
- **动画帧**：待机(2帧循环)、行走(4帧)、攻击(3帧)、防御(2帧)、受伤(2帧)、倒地(1帧)
- **场景背景**：程序化绘制城市废墟，包含远景建筑剪影、中景废墟、近景地面
- **特效**：攻击火花（粒子系统）、防御光盾（半透明像素圆弧）、伤害数字（像素字体渲染）
